# Security Policy

## Reporting Security Issues

Please report security issues privately to the project maintainer. Do not include sensitive information in public issues.

If you need to share details, include only the minimum information needed to reproduce the problem. Do not post private DNS settings, local configuration files, tokens, cookies, credentials, or personal network information.

## Supported Versions

Supported versions can be listed here after official releases are published.

## Local Configuration

The app stores local settings in `config.json` next to the executable. Do not commit real local configuration files that may contain private DNS settings or personal information.

## Scope

This project has not undergone a formal third-party security audit. Review the source code before building or running it in sensitive environments.
